# Lucky3 — Session 留存優化企劃
> 版本：1.0　日期：2026-04-25　作者：產品策略

---

## 一、問題定義

### 現況
Lucky3 的畫廊碎片掉落系統已有完整的「今日進度」設計：

| 今日第幾勝 | 掉落機率 |
|-----------|--------|
| 第 1 勝   | 0%（暖身局） |
| 第 2 勝   | 70% |
| 第 3 勝   | 90% |
| 第 4 勝+  | 100% 保底 |

**這個設計本身就是完美的三局 Session 容器。**

### 核心問題
**玩家看不見這個進度。**

玩家贏完第一局只知道「我贏了」，不知道：
- 今天已經贏了幾局
- 再玩一局有 70% 機率拿到碎片
- 焦點畫作還差幾片就能完成

缺乏可見進度 → 沒有「再玩一局」的心理驅動 → Session 過短 → 留存率低

---

## 二、設計目標

| 指標 | 現況（估）| 目標 |
|------|---------|------|
| 平均每日 Session 時長 | ~3–4 分鐘（1–2 局） | ~11 分鐘（3 局） |
| 次日留存 | 基準 | +15–20% |
| 第 7 日留存 | 基準 | +10% |
| 每日完成三局玩家比例 | ~20% | 待觀察（上線 2 週後定基準） |

---

## 三、設計原則

1. **不改核心規則** — 掉落邏輯、機率、種子池皆不動
2. **只改資訊可見性** — 把已存在的進度顯示出來
3. **不增加焦慮** — 顯示進度是「誘惑」，不是「壓力」
4. **碎片是今日最大的爽點** — 一切視覺設計圍繞這個時刻

---

## 四、功能規劃（P0–P2）

---

### P0：Win 畫面加入「今日進度帶」

**位置：** `showWinPanel()` 與 `showZeroClearWinPanel()` 內，勝利標題下方

**UI 設計：**

```
╔══════════════════════════════════════╗
║   LUCKY 3 JACKPOT                    ║
║                                      ║
║   ● ● ○  今日第 2 勝                 ║
║   再贏一局 → 90% 獲得畫廊碎片 ✦      ║
╚══════════════════════════════════════╝
```

- 三個圓點代表今日三局進度（● 完成 / ○ 未完成）
- 第 1 勝：不顯示碎片提示，只顯示進度點
- 第 2 勝：顯示「再贏一局 → 90% 碎片」
- 第 3 勝：顯示「今日碎片全收！」+ 已掉落的碎片縮圖
- 第 4 勝+：若有保底，顯示「今日保底碎片已獲得」

**資料來源：** `loadDailyWins().count` + `computeDropSpec()` — 兩個函式已存在，直接讀取。

**實作量：** 小（Win Panel HTML 增加一個 div，CSS 加三個樣式）

---

### P0：主畫面焦點畫作 Widget

**位置：** 主畫面底部，Play 按鈕上方（Settings 面板外）

**UI 設計：**

```
┌─────────────────────────────────────┐
│  [縮圖 40×40]  《晨霧山嵐》          │
│                ████████░░  7 / 12   │
└─────────────────────────────────────┘
```

- 顯示目前 `focusPaintingId` 對應畫作的封面縮圖（`cover.jpg`）
- 進度條顯示已收集碎片數 / 總碎片數
- 點擊直接進入畫廊詳細頁
- 完成時縮圖加上金框閃爍，提示切換到下一幅

**狀態更新時機：**
- App 啟動時
- `onWinDropFragment()` 執行後（掉碎片時立即更新）

**資料來源：** `loadGalleryState()` + `PAINTINGS` 陣列 — 全部已存在。

**實作量：** 中（新增 Widget DOM、CSS、更新時機 Hook）

---

### P1：碎片飛入動畫

**觸發時機：** `showFragmentToast()` 執行後

**動畫描述：**
1. 碎片縮圖在畫面中央放大出現（0.3 秒）
2. 停頓 0.4 秒（讓玩家看清楚是哪一片）
3. 碎片縮圖飛向畫面底部的焦點 Widget（0.5 秒 ease-in）
4. Widget 進度條更新 + 微震動
5. 若此片完成整幅畫 → 進入「揭幕儀式」（見 P1b）

**技術方案：** CSS `transform` + `animation`，計算 Widget 位置作為飛行終點。
不需要 Pixi.js，純 CSS 動畫可達成。

**效果：** 讓「拿到碎片」成為整個 Session 最有記憶點的時刻。

---

### P1b：畫作完成揭幕儀式

**觸發：** `galleryUiState.frameRevealPaintingId` 被設定時

**動畫序列：**
1. 畫面轉黑（0.3 秒）
2. 畫作縮圖從中央展開（類似翻開相框）
3. 金色粒子效果（已有 ParticleSystem.emit 可用）
4. 顯示全幅畫作 + 畫作標題
5. 「第 X 幅收藏完成」文字淡入
6. 按鈕：「繼續」

**這是整個收藏系統的最高潮時刻，需要對應的視覺衝擊。**

---

### P2：連勝火焰強化

**現況：** 已有 `streak7` 成就，但視覺不夠顯眼。

**強化方案：** 在主畫面玩家資料區（或 Win 畫面）顯示連勝天數火焰圖示：

```
🔥 × 3   連續 3 天勝利
```

- 1–2 天：小火苗（灰色）
- 3–6 天：橘色火焰
- 7+ 天：金色火焰 + 閃爍

**心理機制：** 玩家不願意看著火焰熄滅，自然形成每日回訪動機。

---

### P2：「今日任務」輕量版

不做完整任務系統，只在 Win 畫面加一行情境提示：

| 情況 | 提示文字 |
|------|---------|
| 今日首勝 | 「明日連勝 +1 🔥」 |
| 今日第 2 勝且有 Combo | 「組合技大師！」 |
| 今日第 3 勝 | 「今日收藏任務完成 ✦」 |
| 達成 zero-clear | 「傳奇清空！你是今日最強判官」 |

純文字、無需新資料結構，只需 i18n 新增幾條 key。

## 六、各功能實作細節

### Win 畫面進度帶（P0）

在 `showWinPanel()` 的 `overlay.innerHTML` 中，勝利標題後加入：

```javascript
// 讀取今日勝場和下一局掉落預期
const dw = loadDailyWins();
const winCountAfter = dw.count;  // onWinDropFragment 已記錄，此時已+1
const nextSpec = computeDropSpec(winCountAfter + 1, loadGalleryState().pityNoDrop || 0);

const dots = [1,2,3].map(i =>
  `<span class="today-dot ${i <= winCountAfter ? 'filled' : ''}"></span>`
).join('');

const nextHint = nextSpec.canDrop
  ? `再贏一局 → ${Math.round(nextSpec.rate * 100)}% 獲得碎片`
  : (winCountAfter >= 3 ? '今日碎片任務完成 ✦' : '');

const progressHtml = `
  <div class="win-today-progress">
    <div class="win-today-dots">${dots}</div>
    <div class="win-today-hint">${nextHint}</div>
  </div>`;
```

### 焦點 Widget 更新時機（P0）

在 `onWinDropFragment()` 末尾加一行：
```javascript
updateFocusWidget();  // 新函式，讀取 loadGalleryState() 並更新 DOM
```

在 `initApp()` 時也呼叫一次 `updateFocusWidget()`。

## 七、優先順序與時程

| 優先 | 功能 | 預估工時 | 預期影響 |
|------|------|---------|---------|
| **P0** | Win 畫面今日進度帶 | 2 小時 | 最直接拉動第 2、3 局 |
| **P0** | 主畫面焦點 Widget | 4 小時 | 每次開 App 的拉力 |
| **P1** | 碎片飛入動畫 | 3 小時 | 提升「拿到碎片」的爽感記憶 |
| **P1b** | 畫作揭幕儀式 | 4 小時 | 完成 12 幅畫的終極高潮 |
| **P2** | 連勝火焰強化 | 2 小時 | 跨日留存 |
| **P2** | 今日任務輕量提示 | 1 小時 | 情境化鼓勵 |

**建議執行順序：** P0 兩項先上，觀察 2 週數據後再做 P1。

---

## 八、不做的事（邊界）

- **不加 Lives / Energy 系統** — 破壞「無套路」品牌承諾
- **不加 Push Notification** — 用拉力取代推力
- **不改掉落機率** — 現有機率已經設計良好
- **不加付費加速碎片** — 維持買斷 / 無廣告定位
- **不增加畫作數量** — 12 幅已足夠，先完成現有體驗閉環

---

## 九、成功驗證方式

上線後觀察 LocalStorage 資料：
- `lucky3-daily-wins-v2` — 每日勝場分布（1局 / 2局 / 3局+）
- `lucky3-gallery-v2` — 碎片收集速度（間接反映回訪頻率）
- `lucky3-win-history-v1` — 連勝天數分布

**目標：** 2 週後「今日 3 勝以上」玩家比例從 ~20% 升至 ~45%。
